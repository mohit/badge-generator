API Surface

Public verification and authenticated issuance

Public routes under /public/api/* support no-key verification and trust discovery. Authenticated routes under /api/* handle issuance, signing, and admin operations.

Issue and verify quickstart

Minimal sequence from domain readiness to public verification.

1. Verify issuer domain

POST /public/api/issuers/verify

2. Create issuer + badge class

POST /api/issuer
POST /api/badge-class

3. Issue + sign

POST /api/credential-subject
POST /api/sign-badge

4. Verify publicly

GET /public/api/verify/badge/:badgeUrl(*)

Implementation-specific additions

Distinct behavior in this implementation beyond baseline issuance/verification endpoints.

Security

SSRF-hardened public verifier routes

Public verify endpoints block localhost/private-network fetch targets before server-side retrieval.

Trust policy

Domain ownership verification flow

Trust writes are tied to issuer metadata hosted at /.well-known/openbadges-issuer.json with rate limits and cooldowns.

Agent-native ops

API + CLI + MCP + llms.txt

Human and LLM workflows can issue, verify, and inspect trust state through the same public contract.

Public endpoint groups

Verification plane
  • GET /public/api/verify/badge/:badgeUrl(*) Fetch and verify a remote badge URL.
  • GET /public/api/verify/issuer/:issuerUrl(*) Validate issuer profile and key discoverability.
  • POST /public/api/verify/json Verify inline badge JSON object.
Trust plane
  • POST /public/api/issuers/verify Rate-limited trust write by domain ownership proof.
  • GET /public/api/trust/issuer/:domain Read issuer trust state record.
  • GET /public/api/trust/events/:domain Read recent trust events for a domain.
  • GET /public/api/trust/issuers?status=verified List trusted issuers by status filter.
